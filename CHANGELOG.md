# Changelog

## 0.3.0

Python版の開発を中止し、**Web版を唯一の実装**とした。
8つの新規検出ルール、5つの新規自動修正を追加。

### 方針転換: Web-only

- Python CLI の開発を中止
- Web UI（単一 HTML / JSX）を唯一の開発・配布対象とする
- 形態素解析依存の4ルール（DR-30〜33）は Web 未移植のまま保留

### 検出ルール: 26 → 34（Web実装）

| ID | 検出内容 | カテゴリ |
|------|------|------|
| DR-39 | 文体混在（です/ます × だ/である） | structural |
| DR-40 | 思考保留表現の多用（と考えられる、と思われる） | structural |
| DR-41 | 強調語の過密（極めて、非常に、大変） | structural |
| DR-42 | 視点（人称）の混在（我々 × あなた） | structural |
| DR-43 | 全角半角混在（１→1、Ａ→A） | visual |
| DR-44 | 数字表記の揺れ（3つ × 三つ） | visual |
| DR-45 | 記号混在（～ × 〜、... × …） | visual |
| DR-46 | 疑問＋断定の不自然な近接 | structural |

## 0.2.0

### 検出ルール: 25 → 30

5つの新規ルールを追加し、未カバーだったAI文体パターンを網羅した。

| ID | 検出内容 | カテゴリ |
|------|------|------|
| DR-34 | カタカナビジネス用語の集中 | structural |
| DR-35 | 列挙マーカーの機械的使用（第一に/第二に…） | structural |
| DR-36 | 文頭パターンの反復（また、…また、…） | structural |
| DR-37 | 名詞化冗長表現（検討を行う、改善を図る） | structural |
| DR-38 | 冗長定型句（〜することが可能です） | structural |

### URL入力対応

CLI の全サブコマンド（scan/fix/report/prompt）で URL を直接指定可能に。

- BeautifulSoup 優先、stdlib HTMLParser フォールバック
- `<article>` / `<main>` を優先抽出
- 文字コード自動検出（Content-Type → meta → utf-8）

### Web UI

ブラウザで解析・修正・プロンプト生成を完結できるインターフェース。

- 3タブ構成: 解析 / 修正 / プロンプト
- 30ルールのうち形態素解析を除く26ルールをJavaScriptに移植
- 6種の自動修正（3層制御）をJS移植
- 14種の構造・文体修正プロンプト生成
- Before/After スコア比較、diff表示
- サーバ不要、単一HTMLファイルで動作

### テスト

- 88 → 97テスト（DR-34〜38 の9テスト + fetcher 15テスト含む）
